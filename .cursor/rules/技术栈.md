## 项目概览

- **框架**：Nuxt `3.10.3`（Vue `^3.4.0`）
- **语言**：TypeScript `^5.3.0`（项目为 ESM：`package.json` 内 `type: "module"`）
- **状态管理**：Pinia `^3.0.4` + `@pinia/nuxt` `^0.11.3`
- **国际化**：
  - 依赖中包含 `@nuxtjs/i18n` `^8.3.0`
  - 但当前代码实际使用 **自定义** `composables/useI18n.ts` + `i18n/language.json`
- **样式**：Stylus（`lang="stylus"`）+ `assets/styles/variables.styl`、`assets/styles/mixins.styl`
- **基础样式**：`normalize.css`

## 运行环境与包管理

- **包管理器**：npm（仓库存在 `package-lock.json`，默认以 npm 为准）
- **Node 版本**：
  - **推荐**：Node.js **20 LTS**（稳定、生态兼容性好）
  - **最低**：Node.js **18+**（需满足 Nuxt 3.10 的运行要求；若实际运行遇到兼容问题，优先升级到 Node 20）

### 常用命令（以 `package.json/scripts` 为准）

- **开发**：`npm run dev`
- **构建（SSR）**：`npm run build`
- **本地预览（SSR 构建产物）**：`npm run preview`
- **生成静态站点（SSG）**：`npm run generate`



## 目录结构约定（AI 生成代码必须遵守）

### 路由与页面

- **页面**：放在 `pages/` 下，遵循 Nuxt 文件路由约定。
  - 示例：`pages/home/index.vue`
- 若新增内容页：优先 `pages/<section>/<slug>.vue` 或 `pages/<section>/index.vue`，保持语义清晰。

### 布局

- **布局**：放在 `layouts/` 下。
  - 示例：`layouts/default.vue`

### 组件

- **共享组件**：放在 `components/shared/` 下（用于 Header/Footer 等全局复用）。
  - 示例：`components/shared/AppHeader.vue`、`components/shared/AppFooter.vue`
- 组件命名：**PascalCase**（如 `AppHeader.vue`）。

### 组合式逻辑（composables）

- 复用逻辑放在 `composables/` 下，命名为 `useXxx.ts`。
  - 示例：`composables/useI18n.ts`

### 状态管理（Pinia）

- 当前仓库尚未出现 `stores/` 目录；如后续引入，请统一使用 `stores/` 并按业务域拆分（例如 `stores/auth.ts`、`stores/content.ts`）。
- 约束：AI **不要**在项目里混用多种状态方案（Vuex、全局单例等）；状态统一用 Pinia。

## Vue/Nuxt 编码约定（AI 生成代码必须遵守）

### Vue SFC 默认写法

- 默认使用：
  - `<script setup lang="ts">`
  - 组合式 API
- 约束：除非有明确理由，不使用 Options API。

### 路径别名

- 在 Nuxt 内使用 `~/` 指向项目根（示例：`~/i18n/language.json`）。
- 约束：避免写相对路径穿越过深（`../../..`）；优先 `~/`。
